# 要件定義

## プロジェクト概要

旅行の日程・スケジュールを管理するシングルページWebアプリケーション。

## 機能要件

### イベント管理
- 日付・時刻・内容を入力してイベントを追加できる
- 時刻は任意入力（未入力の場合は「—」表示）
- イベントを削除できる
- すべてのイベントを一括削除できる

### 表示
- イベントを日付ごとにグループ化してテーブル形式で表示する
- 複数日にわたる旅行に対応し「1日目・2日目…」とラベル表示する
- 日付・時刻順に自動ソートされる
- 旅行タイトルを設定できる

### データ保存
- ブラウザの localStorage にデータを自動保存する
- ブラウザを閉じてもデータが維持される

### 印刷・出力
- 印刷・PDF出力に対応する
- 印刷時はフォームなどのUI要素を非表示にする

## 非機能要件

- ビルド不要（`index.html` 単体で動作）
- 外部依存なし（Tailwind CSS は CDN 経由）
- サーバー不要（ブラウザで直接開いて使用可能）

## 技術スタック

| 項目 | 内容 |
|------|------|
| フロントエンド | HTML / Vanilla JS |
| スタイリング | Tailwind CSS (CDN) |
| データ保存 | localStorage |
| ホスティング | GitHub Pages |
